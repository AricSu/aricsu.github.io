## 产品与架构

- **TIHC**：一个面向 TiDB troubleshooting 的 Chat 系统（以“不落库优先”为默认原则）。
- **Extension / Sidepanel**：浏览器扩展与侧边栏 UI，用于交互与展示流式输出。
- **Serverless**：无服务器运行形态（例如 Vercel Functions），用于承载 API 与编排逻辑。
- **Engine / Chat Engine**：一次请求选择的“执行路径/能力组合”（例如 `openai` / `openai_rag` / `manus` / `tidb`）。

## RAG / 检索增强

- **RAG (Retrieval-Augmented Generation)**：先检索上下文，再把上下文交给模型生成回答的模式。
- **GraphRAG**：用图结构组织/检索上下文的 RAG 方式（常见于文档与知识图谱场景）。
- **Embedding**：将文本映射为向量，用于相似度检索。
- **Chunk**：将长文切分成更小片段以做向量化与检索。
- **Retriever**：负责召回候选上下文的组件。
- **Reranker**：对候选上下文进行二次排序/过滤的组件。

## 隐私与数据

- **Stateless**：默认不存储用户对话内容/上下文（仅做“请求→处理→返回”）。
- **PII**：可识别个人身份的信息（如邮箱、姓名、手机号等）。
- **TTL**：数据过期策略（只在必须落库时建议启用）。
